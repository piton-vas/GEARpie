#### Fixed (соответствие VDI 2736: единое число циклов NL для обеих пар)
- **Рассогласование расчётного ресурса между парами зацепления устранено.** Внешняя пара (`VDI2736.LCC`) жёстко брала NL = 1e8, внутренняя пара (`engine`, бывш. `_PR_CYCLES`) — NL = 1e6; `sh_pessimistic` для смешанных пар тоже считал σHlim при 1e6, хотя SH_код был при 1e8. Для S-N материалов (POM/PA66/капролон PA6_CAST) σ-предел при 1e6 на ~60 % выше, чем при 1e8 (проверено: σHlim венца капролон 78.6 МПа при 1e6 против 48.6 при 1e8 → SH завышался на 62 %). Один и тот же зуб саттелита оценивался при 1e8 (внешняя пара) и 1e6 (внутренняя) — внутреннее противоречие, не соответствующее VDI 2736 (S-N точка должна быть единой для расчётного ресурса коробки). На материалы с константным σ (печатные пластики, POM_C, металлы) не влияло.
- **Число циклов вынесено из констант в `limits.cycles` (NL) и применяется единообразно к ОБЕИМ парам.** `engine.Config.cycles` (по умолч. `DESIGN_CYCLES_DEFAULT = 1e8`); протянуто в `VDI2736.LCC(..., NL=cfg.cycles)` (новый необязательный параметр, дефолт 1e8 — обратная совместимость со всеми существующими вызовами GEARpie), в `planet_ring_check(..., cycles=cfg.cycles)` и в `sh_pessimistic(..., cycles=cfg.cycles)`. Парсинг `spec._parse_cycles` (принимает `limits.cycles` или `limits.NL`, должно быть > 0). В `physics_hash` входит только при `≠ 1e8` (как α и смазка — дефолтный кэш совместим). Сериализуется в `cfg.cycles` кэша и в `load` отчёта. Численно проверено: при едином NL обе пары дают согласованные σ-пределы (1e8: внешн SH 0.823 / внутр SH 0.592; 1e6: 1.381 / 0.987).
- **Документация (SKILL.md):** новый раздел «Расчётный ресурс (`limits.cycles`)»; формат ввода и таблица материалов отражают единый NL; ориентиры (низкая нагрузка ≈1e6, умеренная ≈1e7, промышленная ≈1e8).

#### Audit (сверка расчётов с VDI 2736 / ISO 6336)
- **Проведён аудит всех расчётов на соответствие VDI 2736** (с веб-сверкой первоисточников). Подтверждены корректными: изгиб σF = Ft/(b·m)·YFa·YSa·Yε·Yβ·KF с σFG = σFlim·YST (YST = 2.0 — подтверждено: «VDI applies stress correction factor YST from reference test gears, YST=2.0»); контакт σH с √KH; применение KA (изгиб ×KA, контакт ×√KA — проверено численно: σF×1.300, σH×1.140 = √1.3, двойного учёта нет; «VDI applies load factors K»); температура зуба θ = θ0 + Pvzp·(kθ/(b·z·(vt·m)^0.75) + RλG/A) с kθ 6300/9000 (фланк) и 895/2148 (корень), степень 0.75 (лит-ра: kθ корня ≈2100, степень 0.75); прогиб λ = 7.5·Ft/(b·cosβ)·(1/E1+1/E2) ≤ 0.07·m при НОМИНАЛЬНОЙ Ft (сверено точно по MDPI/VDI 2736); Hertz вогнутой внутренней пары (1/ρ = 1/ρ_сат − 1/ρ_венец); износ Wm (размерно мм). Единственное найденное несоответствие — рассогласование NL (исправлено выше). К сведению (не баги): изгиб внутренней пары по Lewis упрощён (YST сокращается, нет YSa — на нетто ~18 % консервативнее); коэффициент Kγ неравномерности нагрузки между саттелитами не вводится (для податливых пластиковых саттелитов допустимо).

#### Added (смазка в скиле planetary-gearbox)
- **Параметр смазки `limits.lubrication` = `dry` | `grease` | `oil` (по умолчанию `grease`).** Задаёт коэффициент трения μ в формуле нагрева VDI 2736 (нагрев θ−θ0 ∝ μ — подтверждено лит-рой: «пересчёт θ с уменьшенным μ значительно улучшает совпадение с измерениями») и коэффициент износа kw. `engine.Config.lubrication`; пробрасывается в обе пары.
  - **μ с учётом смазки** (`engine.mesh_friction` = μ_сухой × множитель): множители `LUBE_FRICTION_FACTOR` = {dry 1.0, grease 0.40, oil 0.25} — консервативный край измеренного снижения трения смазкой (−60 % grease, −75 % oil; в лит-ре до −87 % при высоком PV). Сухой μ — прежняя таблица `dry_cof` (= `CONTACT.HERTZ`).
  - **Внешняя пара (солнце–саттелит): μ заходит в пайплайн через новый `CONTACT.HERTZ(..., cof_override=μ)`** — `VDI2736.LCC` сам считает TF/TR (а значит σ-пределы и SF/SH) сразу при выбранной смазке, без пост-коррекции (θ−θ0 строго ∝ μ). Обратносовместимо: `cof_override=None` (GEARpie и старые вызовы) → прежняя сухая таблица.
  - **Эффект (капролон `thigh3-CAST-79-D120`, S-N материал):** dry → grease → oil даёт θ_зуба 83.6 → 48.4 → 39.6 °C, SH 1.477 → 1.638 → 1.667, SF 2.04 → 2.17 → 2.21. У S-N материалов (POM/PA66/капролон) смазка реально поднимает запасы через снижение температуры; у константных σ материалов (печатные пластики, POM_C, алюминий) меняются только θ и износ.
- **kw износа по классу пары × смазка** (`engine.wear_coefficient`, заменил константу `WEAR_KW=1e-6`). В VDI 2736 kw есть только для POM/сталь и PBT/сталь (всухую), поэтому остальное — ОЦЕНКА из лит-ры: базовый сухой kw = {углеволокно 2e-6; пластик/металл 3e-6 (≈ POM/сталь, VDI-диапазон 1–5e-6); POM/POM 6e-6; полиамид/прочее пластик-пластик 1e-5} × `LUBE_WEAR_FACTOR` {dry 1.0, grease 0.15, oil 0.05} (снижение износа смазкой ~−85/−95 %). Публикуется в `wear.kw_mm3_per_Nm`; сводная строка `--pretty` показывает kw и тип смазки.
- **Предупреждение: консистентная смазка + полиамид при низком PV.** Для PA-семейства под grease при PV ≈ pm·vt < 10 МПа·м/с (pm = σH·π/4, vt — окружная на делительной; оценка) — смазка может НЕ снижать (а у полиамидов даже увеличивать) износ (лит-ра по PA66, pin-on-disc: при PV 4 МПа·м/с износ ×18 к сухому). Уходит в `model_warnings`/`safety.warnings`; совет — испытание либо масло.

#### Changed (приведение к букве VDI 2736)
- **Прогиб зуба считается по НОМИНАЛЬНОЙ Ft** (раньше — пиковая Ft·Ka). VDI 2736-2 определяет λ = 7.5·Ft/(b·cosβ)·(1/E₁+1/E₂) при номинальной окружной силе; множитель Ka убран. Передаточное выводится из зубьев (`i = 1 + z_ring/z_sun`), задавать не нужно. Каталог моторов (`M3v3`: 3.3 Н·м / 150 об/мин).

Это число идёт в расчёт прочности (больше саттелитов → нагрузка делится → выше запасы) и в вывод (`stages[].n_planets/n_planets_max/n_planets_auto`, `dimensions.by_stage`, строка «Саттелиты» в `--pretty`). Подбор воспроизводит ручные значения колена (z18-27-72→3, z28-28-84→4); при снижении порога до 0.3 мм z20-40-100 берёт 4 саттелита вместо 3 и SH 1.21→1.40.
