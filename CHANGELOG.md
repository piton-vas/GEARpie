## [Unreleased]

### Design
- **Бедро (180 Н·м, Ka 1.3) — подобрана компактная трёхступенчатая планетарка `thigh-PA6A-82`** (скил `planetary-gearbox`, режим `output`). Под мотор ~3.3 Н·м (i≈82): ст.1 z18/45/108 m1.25 b26 (3 сат.), ст.2 z24/18/60 m2.0 b28 (6 сат.), ст.3 z27/18/63 m2.5 b40 (6 сат.). **Ø169.5 × L139 мм, весь PA6_ANNEAL (печать, фрезеровки 0), ПРОХОДИТ** VDI 2736: слабое место — контакт солнца выходной ступени SH=1.45 (×1.04 к порогу 1.4), SF=2.66. Сохранено в кэш `designs/b10993f0338bcfb3.json`.
  - **Ключевые находки:** (1) критична выходная ступень (несёт все 180 Н·м), лимит — контакт, не изгиб; (2) 6 саттелитов (мелкий z_planet=18, z_sun≥24) — главный рычаг прочности, но дают низкое i/ступень, поэтому высокое i ставится на вход, где момент минимален; (3) **PA6_ANNEAL по контакту лучше POM_C** (мягче, E=1700 → больше пятно контакта → ниже σH), т.е. фрезеровать POM_C смысла нет; (4) PETG слаб на контакт — для прохода требует крупного модуля и раздувает коробку.
  - **Проверка «фрезернуть солнце ст.3»** — контрпродуктивно: жёсткое солнце (D16T/POM_C/STEEL) поднимает σH 20→22–28 МПа, и пластиковый саттелит перестаёт держать контакт (SH_pess падает до 1.0–1.3 < 1.4) → НЕ проходит. Сохранены в кэш `7d10d88c…` (D16T), `a12a23e1…` (POM_C), `5c3b4041…` (STEEL).
  - **Проверка «фрезеровать и саттелиты выходной ступени» (меньше саттелитов + выше i):** работает только на СТАЛИ. D16T проваливается — жёсткий металл даёт σH=220–260 МПа > σHlim 170 (алюминий по контакту слабее, чем ожидается), при 3 саттелитах нагрузка/зацепление вдвое выше. POM_C при 3 саттелитах не проходит даже на Ø192 (SH_pess 0.76–0.95). Печатный венец против металлического саттелита тоже не держит (PR SH_ring 0.60–0.93) → венец пришлось бы тоже делать стальным. STEEL-выходная (i=5, 3 сат., m1.0–1.5) проходит с большим запасом и крайне компактна (Ø84–120); полный Вариант B (печатный передний поезд + стальная выходная) даёт **Ø120 × L123, i=87.5, PASS** (`df0b647…`). Сталь отклонена пользователем (вес/много фрезеровки внутреннего венца). Итог: остаётся базовый печатный `thigh-PA6A-82`.
  - **Снижение числа саттелитов (меньше подшипников планет).** ст.2: 6→4 саттелита требует компенсации ширины (m2.0 b28→b32, SH 1.67→1.46; при b28 падает до 1.36). ст.3 (несёт все 180 Н·м, сама — слабое место): 6→5 возможно только расширением венца до b48 (SH восстанавливается до 1.451; b45 — впритык 1.405); 6→3 нереально даже при b70 (SH 1.36 < 1.4). 4 саттелита на ст.3 запрещены сборкой ((z_s+z_r)=90 делится на 3/5/6, но не 4). **Принятый финал `thigh-PA6A-82-345`** (`8bf457a4d9f713b4`): ст.1 z18/45/108 m1.25 b26 (3 сат.), ст.2 z24/18/60 m2.0 b32 (4 сат.), ст.3 z27/18/63 m2.5 b48 (5 сат.) — **Ø169.5 × L151 мм, 12 опор планет (было 15), весь PA6_ANNEAL, PASS**, governing SH=1.451 (ст.3, ×1.036). Размен против базы 3/6/6 (L139): −3 подшипника ценой +12 мм длины.

### Fixed
- **MATERIAL_LIBRARY: `STEEL` и `ADI` теперь callable σ-пределы.** Их `SigmaHlim`/`SigmaFlim` хранились константами (int), из-за чего любое использование стали/ADI как солнца или саттелита роняло внешнюю VDI-пару с `TypeError: 'int' object is not callable` (`VDI2736.LCC` вызывает предел как функцию `SigmaFlim1(T, NL)`). Обёрнуты в `lambda temp, cycles: …` по образцу `D16T` — значения не изменились (STEEL 1500/430 — цементируемая сталь грейда MQ по ISO 6336-5; ADI 700/250 — ausferritic SG-чугун, σHlim в норме, σFlim на верхней границе диапазона ~230–250). Теперь STEEL/ADI работают во всех слотах (sun/planet/ring); расчёты с ними как венцом не изменились. Скил `planetary-gearbox` рекламировал эти материалы для любого слота — теперь обещание выполняется.

### Added
- **Скил `planetary-gearbox` — переносимый расчёт планетарной коробки для любого LLM-агента** (`skills/planetary-gearbox/`). Формат агентского скила (SKILL.md + Python-CLI), вендор-нейтрально (Claude, OpenAI и др.): на вход — параметры всей передачи (требуемый `T_out` или мотор, `Ka`, КПД) и каждой ступени (модуль, ширина, числа зубьев солнца/саттелита/венца, материал); на выход — запасы прочности **SF/SH/SH_pess со слабым местом** (`safety.governing`: ступень/зацепление/деталь), габариты (наружный Ø, осевая длина) и проверка геометрии. Структура: `gearbox/engine.py` (ядро), `catalog.py` (материалы/моторы), `spec.py` (схема ввода/вывода + хэш кэша), `library.py` (поиск), `cli.py` (`calc`/`search`/`materials`/`motors`).
  - **Ядро** — обобщение N-ступенчатого движка `spider_thigh.py`: операционная точка (Ka, T_amb, КПД, требования) вынесена в `engine.Config`; физика VDI 2736 не менялась. Режим `motor` воспроизводит кэш колена **число-в-число** (проверено на `i20_5x4_…Z18-27-72_…Z28-28-84_b35x45`: SF/SH/σH/F_t совпали до отображаемой точности).
  - **Два режима нагрузки:** `output` (рекомендуемый) — нагрузка на зубья раскручивается обратно из требуемого `T_out` по передаточным числам (статическое равновесие, без КПД в тракте нагрузки); `motor` (forward) — момент идёт вперёд от мотора-пресета с КПД. Передаточное выводится из зубьев (`i = 1 + z_ring/z_sun`), задавать не нужно. Каталог моторов (`M3v3`: 3.3 Н·м / 150 об/мин).
  - **Кэш и поиск:** одинаковый spec (по физически значимым полям, sha256) не пересчитывается (`designs/<hash>.json`, флаг `--no-cache`); `search` ищет по кэшу готовых коробок (~265 валидных: новые расчёты + мигрированные исторические «колено»/«бедро») с фильтрами `--i/--t-out-min/--d-max/--l-max/--material/--sf-min/--sh-min/--passing` и сортировкой. Кэш `designs/` отслеживается git (см. «Removed»/«Changed» выше).
  - При активации скил показывает доступные материалы (11: STEEL/ADI/D16T/POM/POM_C/PA66/PA6_CF/PA_CF/PA6_PRINT/PA6_ANNEAL/PETG) и формат ввода (см. `SKILL.md`); есть готовый пример `examples/two_stage_pa6cf.json`.
  - **Подкоманда `best` — лучшие готовые коробки по материалам для требуемого выходного момента.** Для `--t-out-min X` возвращает по три лучших на каждый материал/комбинацию (`--per-material N`), по умолчанию из проходящих VDI 2736, ранжирование по Ø (компактность; `--sort sh|sf|l|i|t_out`). Группировка `--group-by materials|primary`, `--include-failing`. Реализация — `library.best_by_material` поверх `search` (без пересчёта). Запасы берутся из сохранённого `T_out` коробки (legacy — motor-режим); для оценки геометрии под другой момент — `calc` в режиме `output`. Вывод `--pretty` группирует чемпионов по материалу в формате `Ø / b1+b2…` (наружный диаметр / ширины по ступеням).
  - **Авто-подбор числа саттелитов.** Если `n_planets` у ступени не задано, движок (`engine.max_planets`/`resolve_planets`) берёт максимум саттелитов, которые влезают: наибольшее `n`, при котором выполняется условие сборки `(z_sun+z_ring) % n == 0` и зазор вершин соседних саттелитов ≥ `planet_clearance_min` (по умолч. 1.5 мм, FFF-практический; настраивается в `limits`). Это число идёт в расчёт прочности (больше саттелитов → нагрузка делится → выше запасы) и в вывод (`stages[].n_planets/n_planets_max/n_planets_auto`, `dimensions.by_stage`, строка «Саттелиты» в `--pretty`). Явное `n_planets` — override (помечается `n_planets_auto:false`, рядом `n_planets_max`). Подбор воспроизводит ручные значения колена (z18-27-72→3, z28-28-84→4); при снижении порога до 0.3 мм z20-40-100 берёт 4 саттелита вместо 3 и SH 1.21→1.40. Хэш кэша считается по фактически использованному (разрешённому) числу саттелитов.
- **spider_thigh: расчёт тазобедренного сустава («бедро»)** — новый скрипт `spider_thigh.py`, обобщённый по числу ступеней (автодетект 2/3/4 ступеней по ключам `i{k}`): для каждой пары «солнце–саттелит» полный пайплайн VDI 2736, для «саттелит–венец» аналитика Lewis+Hertz. Цепочка моментов/скоростей по ступеням, та же геометрическая валидация (соосность/сборка/соседство), отчёты в `REPORT/SPIDER_THIGH/` (+ `_data/` кэш и `COMPARE.txt`). Вход: мотор 3.3 Н·м / 150 об/мин (как у колена); требование `T_out ≥ 160 Н·м`; `D_MAX=None` — без жёсткого лимита по Ø (диаметр показываем, но не отбраковываем — цель оптимизации: минимальные Ø и ширина b).
- **spider_thigh_variants.json + spider_thigh_sweep.py** — хранилище вариантов (55 шт.) и генератор Парето-свипа (развёртка выходной ступени × ранний поезд × монолит POM-C × тонкая сетка модуля выхода g58 с обрезанным ранним поездом). Материалы переиспользуются из библиотеки колена (PA6_PRINT / PA6_ANNEAL / PA6_CF / POM_C / PETG / D16T).
- **Результат первого свипа бедра:** связывающее ограничение — контакт SH выходной ступени; ранние ступени легко проходят (SH 1.7–2.3) и перезаложены по ширине. Минимальный Ø, проходящий VDI 2736 (SH≥1.4, SF≥2.0) при 4 саттелитах и нулевом смещении: **Ø≈186 мм** (выход z18-20-58, m3=3, b3=70, POM-C; SH=1.49). Меньший Ø достижим только ценой бо́льшей ширины b (Парето Ø↔b) либо профильной коррекции/доп. саттелитов.
- **REPORT/SPIDER_THIGH/README.md** — in-repo документ проекта бедра (требования, организация файлов, рабочий цикл «предложить варианты → посчитать», текущий Парето-фронт). Контекст проекта держим в репозитории, а не в отдельной памяти агента.
