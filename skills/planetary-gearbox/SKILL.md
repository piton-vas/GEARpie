# Планетарная коробка передач (VDI 2736)

Переносимый расчётный движок: на вход — параметры передачи и каждой ступени, на
выход — **запасы прочности со слабым местом** и **габариты**. Физика — пайплайн
GEARpie/VDI 2736 (внешняя пара «солнце–саттелит») + аналитика Lewis/Hertz
(внутренняя пара «саттелит–венец»).

## Как вызывать

Движок — это CLI `cli.py` рядом с этим файлом. Вывод по умолчанию — JSON в stdout
(машиночитаемо), флаг `--pretty` даёт сводку для человека.

> **Важно (окружение):** движок импортирует пакет `CLASSES` из корня GEARpie и
> требует numpy/scipy. Запускай интерпретатором проекта:
> - Windows: `.venv\Scripts\python.exe skills\planetary-gearbox\cli.py ...`
> - POSIX:   `.venv/bin/python skills/planetary-gearbox/cli.py ...`
> Ниже для краткости пишется `python`.

```bash
python skills/planetary-gearbox/cli.py materials [--pretty]   # доступные материалы
python skills/planetary-gearbox/cli.py motors    [--pretty]   # моторы-пресеты
python skills/planetary-gearbox/cli.py calc   --in spec.json [--pretty] [--out r.json]
python skills/planetary-gearbox/cli.py calc   --json '{...}'  [--pretty]
echo '{...}' | python skills/planetary-gearbox/cli.py calc --stdin
python skills/planetary-gearbox/cli.py search --i 20 --passing --d-max 180 [--pretty]
python skills/planetary-gearbox/cli.py best   --t-out-min 60 [--pretty]   # лучшие по материалам
```

При активации **покажи пользователю доступные материалы и формат ввода** — запусти
`materials --pretty` и `motors --pretty` (значения живые), либо используй списки ниже.

## Формат ВВОДА (spec, JSON)

```jsonc
{
  "name": "опционально — имя варианта",
  "load": {
    "mode": "output",        // "output" (по умолч.) или "motor"
    "T_out_req": 60,         // [output] требуемый момент на выходе, Н·м
    "motor": "M3v3",         // [motor] id пресета (или задай "T_in")
    "T_in": 3.3,             // [motor] момент мотора, Н·м (альтернатива "motor")
    "n_in": 150,             // опц. входная скорость, об/мин (для кинематики)
    "Ka": 1.3,               // коэф. нагрузки (умножает момент ДО напряжений); по умолч. 1.3
    "eta_stage": 0.9         // КПД одной ступени; по умолч. 0.9
  },
  "limits": {                // всё опционально
    "D_max": 180,            // лимит наружного Ø, мм (null = без лимита)
    "SF_min": 2.0,           // порог приёмки по изгибу (VDI 2736)
    "SH_min": 1.4,           // порог приёмки по контакту (VDI 2736)
    "ring_wall": 4.0,        // припуск делит.Ø→наруж.Ø, мм (backing за корнем = ring_wall−1.25·m)
    "alpha": 20,             // угол зацепления, град (20 станд.; 25 — выше контактная способность)
    "T_amb": 25,             // окружающая температура, °C
    "lubrication": "grease", // "dry" | "grease" | "oil" (по умолч. grease) — задаёт μ
                             //   в нагреве VDI 2736 и kw износа (см. ниже)
    "cycles": 1e7            // расчётный ресурс, циклов нагружения (точка на S-N кривой
  },                         //   σF/σH). ЕДИНЫЙ для обеих пар. По умолч. 1e8. См. ниже.
  "material": "PA6_CF",      // материал по умолчанию для всех слотов (опц.)
  "stages": [                // одна или несколько ступеней
    {
      "m": 1.5,              // модуль, мм
      "b": 35,               // ширина зубчатого венца, мм
      "z_sun": 18,           // зубья солнца
      "z_planet": 27,        // зубья саттелита
      "z_ring": 72,          // зубья венца (внутренний; должно быть > z_planet)
      "n_planets": 3,        // ОПЦ.: опустите — подберём максимум, что влезает (см. ниже)
      "x_sun": 0.0,          // смещение профиля солнца (опц.)
      "x_planet": 0.0,       // смещение профиля саттелита (опц.)
      "x_ring": 0.0,         // смещение профиля венца (опц.)
      "mat_sun": "PA6_CF",   // материал слота (перекрывает stage.material / верхний material)
      "mat_planet": "PA6_CF",
      "mat_ring": "PA6_CF"
    }
    // ... ступень 2, 3, ... (последовательное соединение: водило k → солнце k+1)
  ]
}
```

Минимальный пример (две ступени, всё PA6_CF, от выходного момента, число
саттелитов подбирается автоматически):
```json
{"load":{"mode":"output","T_out_req":60},"limits":{"D_max":180},"material":"PA6_CF",
 "stages":[{"m":1.5,"b":35,"z_sun":18,"z_planet":27,"z_ring":72},
           {"m":2.0,"b":45,"z_sun":28,"z_planet":28,"z_ring":84}]}
```

### Про передаточное число и моменты
- Передаточное **выводится** из зубьев: `i = 1 + z_ring/z_sun` (солнце вход, водило
  выход, венец неподвижен). Задавать `i` не нужно.
- **Режим `output`** (рекомендуемый): нагрузка на зубья каждой ступени получается
  обратной раскруткой требуемого `T_out_req` по передаточным числам (статическое
  равновесие моментов, КПД в тракт нагрузки не вводится). `T_out` гарантирован.
- **Режим `motor`**: момент идёт вперёд от мотора через ступени с КПД; `T_out`
  вычисляется (= `T_in · i · η^N`) и сравнивается с требованием. Совпадает с
  поведением spider_knee/spider_thigh.

### Число саттелитов (подбирается автоматически)
Если `n_planets` у ступени **не задано**, движок берёт **максимум саттелитов,
которые влезают**: наибольшее `n`, при котором (1) выполняется условие сборки
`(z_sun + z_ring) % n == 0` и (2) зазор между вершинами соседних саттелитов
≥ 1.5 мм (практический предел для FFF-печати; настраивается `limits.planet_clearance_min`).
Больше саттелитов → нагрузка делится на большее число зацеплений → выше запасы.
Найденное число идёт в расчёт прочности и в вывод (`stages[].n_planets`,
`n_planets_max`, `n_planets_auto`). Заданное явно `n_planets` уважается как
override (в выводе помечается `n_planets_auto:false`, рядом — `n_planets_max`).

### Расчётный ресурс (`limits.cycles`)
`cycles` (NL) — число циклов нагружения, по которому читается S-N кривая материала
(σFlim/σHlim). **Единый для обеих пар** зацепления (солнце–саттелит и саттелит–венец)
— раньше внешняя пара жёстко брала 1e8, а внутренняя 1e6, что для S-N материалов
(POM/PA66/капролон) завышало запас внутренней пары до ~60 %. По умолчанию **1e8**
(консервативно, как промышленный ресурс). Для S-N материалов σ-предел при 1e6 на
~60 % выше, чем при 1e8, поэтому ресурс **прямо определяет** SF/SH. У материалов с
константным σ (печатные пластики, POM_C, металлы) `cycles` ни на что не влияет.
Ориентир: низкая нагрузка (эпизодическая работа) ≈ 1e6, умеренная ≈ 1e7,
промышленная/непрерывная ≈ 1e8. В хэш кэша входит только при `≠ 1e8`.

### Про Ka
`Ka` — **входной** коэффициент: он умножает номинальный момент **до** расчёта
напряжений, поэтому SF/SH уже включают этот запас. Пороги приёмки (`SF_min`,
`SH_min`) — отдельная величина. Запас на удар закладывают **либо** в `Ka`, **либо**
в повышенный требуемый SF, но не дважды.

## Формат ВЫВОДА (отчёт, JSON)

Ключевое:
- `ratio` — `per_stage` и `total` передаточное.
- `kinematics` — `T_out_Nm`, `n_out_rpm`, `T_in_equiv_Nm`, `eta_est`.
- `dimensions` — `D_outer_mm` (наружный Ø по большему венцу + 2·стенка),
  `L_axial_mm` (Σb + 30 мм/ступень), `by_stage` (в т.ч. `n_planets`,
  `n_planets_max`, `n_planets_auto` по каждой ступени).
- `safety`:
  - `SF_min`, `SH_min`, `SH_pess_min` — каждый `{value, at:{stage,mesh,member}}`;
  - `governing` — **слабое место**: метрика с наименьшим относительным запасом;
  - `pass` (по порогам SF/SH), `warnings` — геометрические + модельные
    (b/m > 12, прогиб зуба, температура > 120 °C).
- `geometry` — `valid`, `errors`, `warnings` (соосность/сборка/соседство).
- `requirements` — `D_ok`, `T_out_ok`.
- `pass` (верхнего уровня) — прошло ли всё (прочность + Ø + T_out).
- `stages[]` — подробно по каждой ступени: число саттелитов (`n_planets`,
  `n_planets_max`, `n_planets_auto`), σF/σH и SF/SH для пар sun-planet и
  planet-ring, диаметры, материалы; плюс:
  - `b_over_m` — отношение ширины к модулю (применимость VDI: ≤ 12);
  - `deflection` — прогиб зуба `{lambda_sun_planet_mm, lambda_planet_ring_mm,
    limit_mm (=0.07·m), ok}`;
  - `sun_planet.tooth_temp_C` — температуры зуба VDI 2736 `{flank:[sun,planet],
    root:[sun,planet]}` (использованы в σ-пределах);
  - `sun_planet.wear` — средний износ VDI 2736-2 `{W_mean_mm [sun,planet]
    (за NL_ref=1e6 циклов своего колеса), W_limit_mm (=0.2·m),
    N_to_limit (оценка ресурса до предела), Hv, kw_mm3_per_Nm}`;
  - `planet_ring.tooth_temp_C` — `{flank:{planet,ring}, root:{planet,ring}}`.

`mesh` ∈ {`sun-planet`, `planet-ring`}; `member` ∈ {`sun`,`planet`,`ring`}.

**SH vs SH_pess:** `SH` — как в коде GEARpie (σHP по солнцу). Для смешанной пары
металл–пластик это завышено; `SH_pess` берёт σHlim по более слабому материалу.
Для одинаковых материалов они равны. Для смешанных пар ориентируйся на `SH_pess`.

## Кэш и поиск готовых коробок

- **Кэш:** одинаковый spec (по физически значимым полям) не пересчитывается —
  результат берётся из `designs/<hash>.json` (`from_cache: true`). Отключить:
  `--no-cache`; не сохранять: `--no-save`.
- **Поиск** по уже рассчитанным коробкам (кэш `designs/` — 435 коробок,
  427 геометрически валидных, 70 проходят VDI 2736: новые расчёты + исторические
  наборы «колено»/«бедро», вся библиотека под единым режимом grease + cycles=1e7):
  ```bash
  python .../cli.py search --i 20 --i-tol 0.5 --passing --d-max 180 \
      --material PA6_PRINT --sf-min 2 --sh-min 1.4 --sort d --top 10 --pretty
  ```
  Фильтры: `--i/--i-tol`, `--n-stages`, `--t-out-min`, `--d-max`, `--l-max`,
  `--material` (содержит), `--materials-subset` (только из набора), `--sf-min`,
  `--sh-min`, `--sh-pess-min`, `--passing`. Сортировка `--sort d|l|sh|sf|i|t_out`.
- **Лучшие коробки по материалам** (`best`) — для требуемого выходного момента
  вернуть **по одному чемпиону на каждый материал/комбинацию** среди уже
  рассчитанных. По умолчанию: только проходящие VDI 2736, ранжирование по Ø
  (компактность), 3 на материал.
  ```bash
  python .../cli.py best --t-out-min 60 --pretty            # лидеры на ≥60 Н·м
  python .../cli.py best --t-out-min 160 --per-material 2 --sort sh --pretty
  ```
  Те же фильтры, что у `search`; плюс `--per-material N` (сколько на материал, по умолч. 3),
  `--group-by materials|primary` (полный набор / материал солнца ст.1),
  `--include-failing` (не только проходящие). У каждой записи поле `group` —
  метка материала.

  > Запасы в библиотеке посчитаны при нагрузке самих коробок (исторические
  > «колено»/«бедро» — режим `motor` 3.3 Н·м; `T_out` — достижимый выход).
  > **Вся библиотека пересчитана под единый режим: смазка `grease`, ресурс
  > `cycles = 1e7`** (обе пары при одном NL). Чтобы воспроизвести/расширить запись
  > один-в-один, передавай `limits.cycles: 1e7` (по умолч. `calc` берёт 1e8 —
  > консервативнее, и тогда будет кэш-промах).
  > `best`/`search` фильтруют по сохранённому `T_out`. Чтобы оценить готовую
  > геометрию под ДРУГОЙ целевой момент — пересчитай её через `calc` в режиме `output`.

## Модель прочности (что считается)
- Внешняя пара (солнце–саттелит): полный пайплайн VDI 2736 (геометрия Litvin,
  контакт Герца, температура зуба, σF/σH).
- Внутренняя пара (саттелит–венец): Lewis (изгиб) + Hertz выпукло-вогнутый
  (MAAG/GEARpie не считает внутреннее зацепление). σ-пределы — при температуре
  зуба VDI 2736-2 (нагрев трением: Pvzp = P·Hv·μ, Hv Ohlendorf для внутреннего
  зацепления, μ с учётом смазки; те же kθ, что во внешней паре).
- **Смазка** (`limits.lubrication` = `dry`/`grease`/`oil`, по умолч. `grease`):
  задаёт μ в нагреве (θ−θ0 ∝ μ; множители к сухому μ: dry 1.0, grease 0.40,
  oil 0.25) и kw износа. Смазка снижает θ → у S-N материалов (POM/PA66/капролон)
  растут SF/SH; у материалов с константным σ меняются только θ и износ. Пример
  (капролон): dry→grease θ 84→48 °C, SH 1.48→1.64.
- **Температура зуба** (обе пары) — в выводе (`tooth_temp_C`); S-N данные
  VDI 2736 заданы до 120 °C, выше — предупреждение (пределы клампятся).
  Модель нагрева — для пластиков: чисто металлические пары публикуются при
  T_amb (σ-пределы металлов библиотеки от температуры не зависят).
- **Прогиб зуба** (VDI 2736-2): λ = 7.5·Ft/(b·cosβ)·(1/E₁+1/E₂) ≤ 0.07·m,
  Ft — номинальная (как в VDI); превышение — предупреждение (мягкий зуб
  «проседает»). E — при 25 °C (библиотека не задаёт E(T)); со смазкой зуб
  холоднее → приближение точнее, для горячего сухого зуба λ выше расчётного.
- **b/m ≤ 12** — область применимости упрощения K=KA·KV·KHβ·KHα≈1…1.25;
  более широкие колёса дают предупреждение (реальные SF/SH ниже расчётных,
  ширина работает не вся).
- **Износ** (внешняя пара): средняя формула VDI 2736-2
  Wm = Td·2π·NL·Hv·kw/(b·z·lFl). kw — ОЦЕНКА по классу пары × смазка (VDI даёт
  kw только для POM/сталь всухую): база dry {углеволокно 2e-6, пластик/металл
  3e-6, POM/POM 6e-6, полиамид/прочее 1e-5} × {dry 1.0, grease 0.15, oil 0.05}
  мм³/(Н·м) → `wear.N_to_limit` — оценка циклов колеса до износа 0.2·m.
  Информативно, на `pass` не влияет. Внимание: консистентная смазка + полиамид
  при низком PV может НЕ снижать (и даже увеличивать) износ — отдельное
  предупреждение.
- Прямозубые (β=0), угол профиля из `limits.alpha` (по умолч. 20°; участвует
  и в напряжениях, и в геометрической проверке соосности при x≠0).
- **Расчётный ресурс** `limits.cycles` (NL) — точка на S-N кривой σF/σH, **единая
  для обеих пар** (см. раздел «Расчётный ресурс»). По умолч. 1e8; для S-N
  материалов прямо определяет SF/SH (при 1e6 σ-предел на ~60 % выше, чем при 1e8).
- Геометрия отклоняется (прочность не считается), если нарушены соосность,
  условие сборки `(z_sun+z_ring) % n_planets == 0` или соседство саттелитов;
  εα < 1 в любой паре даёт предупреждение (прерывистое зацепление).
